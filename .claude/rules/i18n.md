# i18n

## All content is translated

No user-facing string is hardcoded. Every label, message, error, and piece of
copy goes through the translation layer, never a literal in the markup or code.

## Controlled vocabularies are codes, not labels

Store controlled-vocabulary values (e.g. sample nature) as stable machine codes
in `lower_snake_case`, never human labels. The code is what the domain schema,
database, and API carry. Labels are translation keys resolved per package: map
code to label with a typed `Record<Code, () => string>` so a new code fails to
compile until it is translated.

## Tests use English

All test content (input data, expected strings, fixtures, sample copy) MUST be
in English. Assert against the English text, not French placeholders.

## Frontend URL localization

The `frontend` (public, indexed) localizes by URL: every localized route carries
a language prefix (e.g. `/fr/...`), and every page declares a
`<link rel="canonical">` to its English (base-locale) version so search engines
rank a single canonical page. `admin` is exempt: authenticated, single locale,
not indexed.
