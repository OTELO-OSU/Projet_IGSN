# i18n

## All content is translated

No user-facing string is hardcoded. Every label, message, error, and piece of
copy goes through the translation layer, never a literal in the markup or code.

## Controlled vocabularies are codes, not labels

Store controlled-vocabulary values (e.g. sample nature) as stable machine codes
in `lower_snake_case`, never human labels. The code is what the domain schema,
database, and API carry.

Every domain enum's translation keys MUST carry the enum name as a prefix:
`<enum>_<code>` (`nature_hand_sample`). App catalogs merge into one namespace,
so an unprefixed key collides; the prefix also ties the key back to its enum. A
code reused under several tree parents shares one key.

## Shared enum translations live in domain

A vocabulary is domain data, so both its text and its resolving logic live in
`domain`, never in `design-system` or an app:

- **Text**: `packages/domain/messages/{locale}.json`, merged into each app via
  its `project.inlang/settings.json` `pathPattern`.
- **Resolver**: written once (e.g. `sample/create-sample-labels.ts`). Paraglide
  compiles a separate `m` per app, so the resolver takes `m` as a parameter;
  each app binds its own `m` in one line (`as unknown as Messages` cast). Never
  copy the resolver per app.

App-only copy backed by no shared enum (page text, form chrome, columns) stays
in that app's catalog.

## Enforce coverage at compile time

A missing translation must fail the build, not render a raw key.

- **Flat vocabularies** (input is a literal union): assert the catalog covers the
  enum, so a missing key fails to compile:

      import type Catalog from "../../messages/en.json" // type-only, erased
      type MessageKey = keyof typeof Catalog
      type AssertKeys<T extends MessageKey> = T // errors if a key is missing
      type _nature = AssertKeys<`nature_${Nature}`>

  A `Record<Code, () => string>` map does the same where the app binds `m`
  directly.

- **Tree vocabularies** (runtime `string` paths): unreachable by types, so cover
  them with a spec walking every path, failing on any label that is its own key.

## Tests use English

All test content (input data, expected strings, fixtures, sample copy) MUST be
in English. Assert against the English text, not French placeholders.

## Frontend URL localization

The `frontend` (public, indexed) localizes by URL: every localized route carries
a language prefix (e.g. `/fr/...`), and every page declares a
`<link rel="canonical">` to its English (base-locale) version so search engines
rank a single canonical page. `admin` is exempt: authenticated, single locale,
not indexed.
